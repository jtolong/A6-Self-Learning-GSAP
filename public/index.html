The Problem I Wanted To Solve

My pages felt static. CSS transitions were fine, but I needed control over timing, chaining, and scroll‑based motion. I also wanted performance that holds up on mobile.

  • Animate with code, not exports
  • Sync motion to scroll position
  • Build timelines I can reason about

What I Learned First

The core pattern is simple: animate properties with .to or .from, then chain those calls in a timeline. ScrollTrigger wires those animations to the page.

  • .to and .from for property animation
  • .timeline to sequence steps
  • ScrollTrigger to pin, scrub, and observe

Pinning keeps an element fixed while content scrolls. Scrubbing ties animation progress to scroll progress. It feels great when used lightly.
